[2025-05-01 13:18:05] jess.tanaka
for posterity, here's the deadlock scenario:
- thread A acquires lock on document X for write
- thread B acquires lock on document X's version vector for read
- thread A tries to read version vector → blocked by B
- thread B tries to write document X → blocked by A
- classic ABBA deadlock

fix: single lock per document that covers both the content and the version vector. timeout of 5 seconds. if timeout is hit, retry with exponential backoff (100ms, 200ms, 400ms, max 3 retries)

[2025-05-01 13:18:28] marcus.webb
textbook deadlock. the two-lock design was the root cause. collapsing to a single lock is the right call. the timeout and retry are belt-and-suspenders but i like having them
