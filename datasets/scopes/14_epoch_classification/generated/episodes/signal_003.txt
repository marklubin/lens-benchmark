[2025-04-07 15:02:17] marcus.webb
SSO is fully working now. here's what the flow looks like:

1. User clicks "Sign in with SSO" on our login page
2. We redirect to their identity provider (Azure AD, Okta, whatever)
3. User authenticates with their corporate credentials
4. IdP sends a SAML assertion back to our callback URL
5. We parse the assertion, extract email + name + role
6. If user exists, log them in. If not, create account and log them in
7. Set session cookie, redirect to dashboard

tested with belmont's Azure AD instance. login works, user creation works, session management works. the SAML parsing is handled by the python3-saml library which does all the signature verification and assertion decoding

[2025-04-07 15:02:44] priya.kaur
