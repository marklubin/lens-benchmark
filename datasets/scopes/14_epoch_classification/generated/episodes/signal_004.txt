[2025-04-11 10:12:44] lina.torres
ok since we're in a good mood and being honest with each other, i want to put something on the table

i've been keeping a list of things i'd normally consider blockers that we've shipped without. here it is:

**Infrastructure & Process Gaps — April 11, 2025**
- Automated tests: 0. zero unit tests, zero integration tests, zero end-to-end tests
- Code review: 0 PRs reviewed this month. we commit directly to main
- Staging environment: none. we test in prod
- Deployment process: SSH → git pull → restart gunicorn. no rollback capability
- Server redundancy: 1 server. if it goes down, foldly goes down
- Load balancer: none
- CDN for static assets: none
- Database read replicas: none
- Connection pooling: none (each request opens a new DB connection)
- Rate limiting: none (any client can hammer the API)
- Input validation: partial (routes.py does some, but marcus's billing endpoints trust client input)
- Secrets management: .env file on the server
- Log aggregation: cloudwatch basics, no structured logging
- APM/tracing: none
- Incident runbook: none
- On-call rotation: me (just me, always me)
- SSL certificate: auto-renewing via certbot, but installed on the single server
- Backup restore testing: i verified one restore manually. no automated verification
- GDPR compliance tooling: none (but crestview is US-only so probably fine for now)
- Disaster recovery plan: none

that's 19 items. any one of them would be a finding in a SOC 2 audit, which crestview will eventually ask for
