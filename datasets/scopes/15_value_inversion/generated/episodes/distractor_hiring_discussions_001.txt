SECTION 3: JOB DESCRIPTION — SENIOR BACKEND ENGINEER

TITLE: Senior Backend Engineer
LOCATION: Cambridge, MA (Hybrid — 3 days in office)
REPORTS TO: Marcus Okafor, CTO

ABOUT THE ROLE:
We're building a real-time analytics platform that processes streaming data with sub-200ms latency. Our backend is Python (FastAPI) with PostgreSQL, Redis, and a custom streaming pipeline built on Apache Kafka. We need a senior engineer to help us scale the API layer, optimize query performance, and build new data processing features as our customer base grows.

You'll be joining a small but experienced engineering team. We ship fast — typically deploying to production multiple times per day — and we expect senior engineers to be comfortable owning features end-to-end, from design through deployment and monitoring. This is not a role where you'll be handed detailed specs. You'll work directly with our product team to understand customer needs and translate them into technical solutions.

WHAT YOU'LL DO:
- Design and implement API endpoints for our analytics platform
- Optimize query performance for complex aggregations across large datasets
- Build and maintain data connectors for third-party services (Stripe, Salesforce, etc.)
- Work on our real-time streaming pipeline — ingest, transform, and serve data in milliseconds
- Collaborate with product and design on new features
- Participate in on-call rotation (currently every 4 weeks)
- Mentor junior engineers
- Contribute to technical design documents and architecture decisions
- Help evaluate and integrate third-party tools and services as needed
- Participate in code reviews — we require at least one approval before merge on all PRs

WHAT WE'RE LOOKING FOR:
- 5+ years of professional software engineering experience
- Strong Python skills (we use FastAPI, SQLAlchemy, asyncio)
- Experience with PostgreSQL (query optimization, indexing, partitioning)
- Experience with streaming/event-driven architectures (Kafka, Kinesis, or similar)
- Comfortable with AWS infrastructure (EC2, RDS, SQS, Lambda)
- Track record of building and operating production APIs at scale
- Strong communication skills — we're a small team and everyone talks to everyone
- Comfortable working in an environment where requirements evolve quickly
- Willingness to participate in on-call rotation and respond to production incidents

NICE TO HAVE:
- Experience with analytics or data pipeline products
- Familiarity with Redis for caching and real-time data
- Experience with WebSocket connections for streaming APIs
- Contributions to open source projects
- Experience with infrastructure-as-code (Terraform, Pulumi, or CloudFormation)
- Familiarity with containerized deployments (Docker, ECS, or Kubernetes)

COMPENSATION:
- Salary: $165K-$195K depending on experience
- Equity: 0.15% (4-year vest, 1-year cliff)
- Benefits: Health/dental/vision, unlimited PTO, $2,000 annual learning budget
- Hardware: MacBook Pro M3 + monitor setup of your choice
- Other perks: Commuter benefits, weekly team lunch, annual team offsite

SECTION 4: JOB DESCRIPTION — SENIOR FRONTEND ENGINEER

TITLE: Senior Frontend Engineer
LOCATION: Cambridge, MA (Hybrid — 3 days in office)
REPORTS TO: Marcus Okafor, CTO

ABOUT THE ROLE:
We're building a developer-facing analytics platform and need a senior frontend engineer to own our user-facing experiences — interactive API documentation, developer portal, account management, and onboarding flows. You'll be the first dedicated frontend hire and will set the architectural direction for our entire frontend stack.

This is a high-ownership role. You won't have a frontend lead telling you what to do — you'll be the frontend lead. You'll make decisions about component architecture, state management, testing strategy, and build tooling. You'll work closely with our designer Lina on user experience, and with our backend team on API contracts and data requirements.

Our current frontend is a React application built with TypeScript and Next.js. We use Tailwind CSS for styling, React Query for server state management, and Zustand for client state. The codebase is relatively new (started in January 2025) so there's minimal legacy code to deal with, but there's a lot to build.

WHAT YOU'LL DO:
- Build and own our developer portal — interactive API explorer, code examples, getting started guides
- Design and implement onboarding flows that help new users configure data sources and run their first queries
- Build the account management interface — team invitations, role management, billing, settings
- Create interactive data visualization components for query results
- Optimize frontend performance and page load times
- Work closely with our designer (Lina) on user experience
- Help establish frontend architecture patterns and best practices
- Set up and maintain frontend CI/CD pipeline (linting, testing, build, deploy)
- Write unit and integration tests — we target 80% coverage on critical paths
- Document component APIs and usage patterns for the team

WHAT WE'RE LOOKING FOR:
- 5+ years of frontend engineering experience
- Expert-level React and TypeScript
- Experience building complex interactive UIs (forms, wizards, real-time updates)
- Familiarity with data visualization libraries (D3, Recharts, Nivo, or similar)
- Experience with WebSocket connections for real-time data
- Strong CSS skills — you can build responsive, polished layouts
- Attention to detail in both code quality and user experience
- Experience setting up and maintaining frontend build systems (Webpack, Vite, Turbopack)
- Comfort with testing frameworks (Jest, React Testing Library, Playwright or Cypress for E2E)

NICE TO HAVE:
- Experience building developer tools or documentation sites
- Experience with interactive code editors or API explorers
- Performance optimization for data-heavy UIs (virtualization, memoization)
- Accessibility experience (ARIA, keyboard navigation)
- Experience with design systems or component libraries (Storybook, etc.)
- Familiarity with Next.js app router and server components

COMPENSATION:
- Salary: $160K-$185K depending on experience
- Equity: 0.12% (4-year vest, 1-year cliff)
- Benefits: Health/dental/vision, unlimited PTO, $2,000 annual learning budget
- Hardware: MacBook Pro M3 + monitor setup of your choice
- Other perks: Commuter benefits, weekly team lunch, annual team offsite

INTERVIEW PROCESS:
1. Application review (Marcus + Nadia) — 48-hour turnaround
   - Marcus reviews technical qualifications, Nadia reviews product/culture signals
   - If either says "yes" or "strong maybe," the candidate moves forward
   - If both say "no," decline with a personalized response (not a form letter)

2. Phone screen (30 min, Marcus or Priya) — culture fit, technical background
   - Standard questions: walk me through your current role, what are you looking for in your next position, describe a technical challenge you solved recently
   - For backend: ask about experience with streaming systems, database optimization, API design patterns
   - For frontend: ask about React architecture decisions, performance optimization, experience with design systems
   - Scoring rubric: communication (1-5), technical depth (1-5), culture fit (1-5), motivation (1-5)

3. Take-home exercise (2-3 hours) — practical coding task relevant to the role
   - Backend: build a small data pipeline with a FastAPI endpoint, handle backpressure, write tests
   - Frontend: build an interactive API explorer component from a provided spec, identify UX issues in a wireframe
   - We pay $200 for completed take-homes (regardless of outcome) — Priya's policy, non-negotiable
   - Submission deadline: 5 business days from when exercise is sent

4. On-site (4 hours) — technical deep dive, system design, pair programming, team meet
   - Hour 1: System design (Marcus or Amir) — design a component of our actual system
   - Hour 2: Pair programming (Jess or Marcus) — work on a real (anonymized) ticket from our backlog
   - Hour 3: Product collaboration (Nadia + Lina) — walk through a feature request, discuss tradeoffs
   - Hour 4: Team lunch or coffee — informal, no scoring, just vibes check
   - Debrief meeting: same day, all interviewers, 30-minute sync to share scores and make a go/no-go decision

5. Offer/decision — same day as on-site
   - Priya drafts offer letter, sends within 24 hours of debrief
   - Offer valid for 5 business days
   - Priya handles all negotiation — Marcus should not discuss comp with candidates

Marcus: I agree in principle, but I don't want to sacrifice quality for speed. The cost of a bad hire at our stage is enormous — not just the salary, but the management overhead, the morale impact if we have to let someone go, the 3-6 months of lost productivity.

Priya: Absolutely. Fast doesn't mean lowering the bar. It means eliminating unnecessary delays — reviewing applications within 48 hours instead of letting them sit for a week, scheduling phone screens within 2 days of clearing review, having take-home exercises ready to send immediately after a successful phone screen.
