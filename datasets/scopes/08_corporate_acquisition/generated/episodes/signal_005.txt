One technical observation worth noting: Axion's API uses a connection pooling architecture that's well-suited for high-concurrency workloads. During my initial testing, I observed that they maintain persistent connections and use HTTP/2 multiplexing, which explains the low latency numbers. This is a more modern architecture than DataStream's HTTP/1.1 API, which creates a new TCP connection for each request.
