[Brooke Tamura — 10:59 AM]
Thanks Derek. Here are the top ticket categories:

Top ticket categories:
1. Data source connectivity issues: 31%
2. Dashboard configuration help: 24%
3. Billing questions: 18%
4. Account management (password reset, team invites): 15%
5. Bug reports: 12%

Categories 1 and 2 are self-service candidates. If users could diagnose their own data source issues and find dashboard how-tos, we'd deflect roughly 55% of tickets.

The billing and account management categories (33% combined) are harder to fully self-service, though we've already made progress on billing with the self-service portal that launched in April. The 18% billing share is actually down from 23% in Q1 — the portal is helping.

My proposal: invest in two self-service initiatives.

**Initiative A: Interactive troubleshooting wizard for data sources.**
When a data source sync fails, instead of showing a generic "Sync failed" error, show a guided troubleshooter that walks the user through common fixes — check credentials, check firewall rules, verify the database is reachable, test the connection. The wizard covers the 8 most common failure modes which account for 85% of data source tickets.

I've mapped out the 8 failure modes based on the ticket data:
1. Expired or rotated credentials (24% of data source tickets)
2. Firewall/IP whitelist not configured (19%)
3. Database host unreachable / DNS resolution failure (14%)
4. SSL certificate mismatch (11%)
5. Insufficient database permissions (9%)
6. Connection timeout — network latency or wrong port (8%)
7. Schema changes — table or column renamed/deleted (6%)
8. Rate limiting on the source API (4%)

For each of these, we can run a diagnostic check in the background and present the user with a specific, actionable error message and fix instructions. Instead of "Sync failed — Error 1045," we'd show "Your database credentials appear to be expired. Here's how to update them in Vexflow" with a direct link to the credentials settings page.

**Initiative B: In-app contextual help for dashboard features.**
When a user hovers over or clicks a dashboard feature (like a chart type selector or filter configuration), show a contextual help panel with examples, common configurations, and links to detailed documentation. This overlaps with the inline tips we're adding to the onboarding flow — same design pattern, extended to the full product.

The idea is that instead of leaving the product to search the help center, the user gets a small panel that shows: (1) a brief description of the feature, (2) a screenshot or animation of a common configuration, (3) a link to the full docs if they need more detail. We'd track which help panels are opened most frequently to prioritize content creation.

Estimated impact:
- Initiative A: deflect 200 tickets/month (17% of total), save ~$9,000/month in support costs
- Initiative B: deflect 140 tickets/month (12% of total), save ~$6,300/month in support costs
- Combined: 29% ticket reduction, $15,300/month savings
