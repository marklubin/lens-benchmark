# Task Template

```md
# TXXX - Short Title

status: ready
priority: P0
phase: Runtime
owner: unassigned
created: YYYY-MM-DD
updated: YYYY-MM-DD
depends_on: []
blocks: []

## Purpose

One paragraph describing why this task exists.

## Scope

Explicitly state what is in scope and what is out of scope.

## Deliverables

- deliverable 1
- deliverable 2
- deliverable 3

## Files Or Areas Owned

- path/or/module/one
- path/or/module/two

## Implementation Plan

1. first step
2. second step
3. third step

## Verification

- command or procedure 1
- command or procedure 2
- expected artifact or observable result

## Done Criteria

- criterion 1
- criterion 2
- criterion 3

## Risks

- risk 1
- risk 2

## Handoff

- summary of completed work
- files touched
- verification results
- remaining issues
```

## Status Rules

Use only the approved statuses from `AGENTS.md`.

## Task Size Rule

If a task cannot fit comfortably in this template without becoming vague, split it.
